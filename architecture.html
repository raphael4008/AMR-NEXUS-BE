AMR-Nexus One Health Backend — Rev 2 Architecture

Mission: A production-ready surveillance platform for Antimicrobial Resistance (AMR) in Kenya, aligned with WHO GLASS, HL7 FHIR, and WOAH standards. This backend repository serves strictly as the Data Backbone and API Gateway.

1. Core Architecture (July 14, 2026 Revision 2)

The system is strictly divided into the internal API gateway (this repository) and external ML microservices (owned by the Data Science team).

Component A: Data Backbone (This Repository)

  • Role: Robust, "real-data ready" schema focused on Human + Poultry data unification. Serves as the central API gateway.
  • Data Entities: AMRRecord (core), Alert (AI triggers), Guidance (LLM advisories), GenomicSignal.
  • Ingestion: POST /api/v1/backbone/ingest/whonet with strict Pydantic validation and data cleaning pipeline (imputing facility_type, sub_county).
  • Dashboard API: Surfaces high-level telemetry, aggregates, and alerts for the frontend UI.

Component B & C: ML Engine & LLM Advisory (External Microservices)

  • Role: Unsupervised hotspot detection, forecasting, and LLM Markdown generation.
  • Decoupling: The AI/ML team reads from the PostgreSQL/Kafka instances, performs the heavy compute, and writes results (Alerts, Guidance) back into the database.
  • Zero Local ML: This backend repository does not contain or execute any ML training, inference, or LLM logic.

2. Monorepo Structure & Separation of Labor

The repository has been restructured into a strict Monorepo to support the 16-person team. Strict boundaries prevent overlap between software engineers, data scientists, and frontend developers.

3. Key Modules & Services

Module File Responsibilities
Application Factory src/main.py FastAPI app instantiation, lifespan events (DB startup checks), CORS configuration, and unified route prefix mounting (/api/v1).
Security src/core/security.py JWT token encoding/decoding, password hashing, and the RoleChecker dependency for role-based access control.
Data Models src/models/entities.py SQLAlchemy ORM definitions: AMRRecord, Alert, Guidance, GenomicSignal.
Data Cleaner src/services/ingestion/cleaner.py Calculates quality scores, flags critical failures (missing pathogen/drug), and imputes missing non-critical fields.
Intelligence API src/api/intelligence.py Fetches database records and alerts to serve aggregated telemetry directly to the frontend dashboards.

5. Test Infrastructure (/backend/tests)

The hermetic test suite (pytest) ensures pipeline integrity without requiring external services.

  • SQLite In-Memory: Uses StaticPool and overrides SessionLocal during testing to isolate tests from Postgres.
  • Comprehensive Coverage:
    • test_api.py: Validates data backbone ingestion and telemetry schemas.
    • test_cleaner.py: Verifies data quality scoring and imputation logic.
    • test_security.py: Multi-role RBAC validation.
