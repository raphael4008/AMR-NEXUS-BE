AMR-Nexus One Health Backend — Rev 2 Architecture

Mission: A production-ready surveillance platform for Antimicrobial Resistance (AMR) in Kenya, aligned with WHO GLASS, HL7 FHIR, and WOAH standards. Transitioned from a simple data ingestion pipe into a dynamically contextualized Decision-Support, Adaptive Intelligence, and Early Warning platform.

1. Core Architecture (July 14, 2026 Revision 2)

The system is consolidated into 3 High-Performance Tightly Integrated Components:

Component A: Data Backbone (The Foundation)

  • Role: Robust, "real-data ready" schema focused on Human + Poultry data unification.
  • Data Entities: AMRRecord (core), Alert (AI triggers), Guidance (LLM advisories), GenomicSignal.
  • Ingestion: POST /api/v1/backbone/ingest/whonet with strict Pydantic validation and data cleaning pipeline (imputing facility_type, sub_county).

Component B: AI Early Warning Engine (The Detector)

  • Role: Unsupervised hotspot detection and time-series forecasting.
  • Anomaly Detection: Uses IsolationForest via AMRAnomalyEngine to score records and generate Alert records (with hotspot_magnitude and anomaly_score).
  • Forecasting: On-demand Prophet-based projection via GET /api/v1/intelligence/forecasts/{county}.
  • Notification: Asynchronous SMS delivery via Africa's Talking (NotificationService) upon high-magnitude alerts.

Component C: Adaptive Intelligence / LLM Advisory (The Advisor)

  • Role: Generates actionable, role-gated markdown advisories using Claude 4.5 Sonnet.
  • Generation: POST /api/v1/intelligence/alerts/{alert_id}/advisory triggers the LLMAdvisoryEngine.
  • Role-Gating: System prompts differ based on user role (e.g., National Coordinator vs. County Veterinarian), ensuring contextually appropriate clinical and policy guidance.
  • Graceful Degradation: LLM API failures persist a PENDING guidance record for retry, ensuring no data loss.

3. Key Modules & Services

Module File Responsibilities
Application Factory src/main.py FastAPI app instantiation, lifespan events (DB startup checks), CORS configuration, and unified route prefix mounting (/api/v1).
Security src/core/security.py JWT token encoding/decoding, password hashing, and the RoleChecker dependency for role-based access control.
Data Models src/models/entities.py SQLAlchemy ORM definitions: AMRRecord, Alert, Guidance, GenomicSignal.
Data Cleaner src/services/ingestion/cleaner.py Calculates quality scores, flags critical failures (missing pathogen/drug), and imputes missing non-critical fields.
LLM Advisory src/services/intelligence/llm_advisory.py Compiles context (SHAP, geography, resistance metrics) and queries Anthropic Claude to generate Markdown briefs.
Forecaster src/services/ml_engine/forecaster.py Uses Facebook Prophet to forecast resistance trends on a per-county basis.

5. Test Infrastructure (120/120 Passing)

The hermetic test suite (pytest) ensures pipeline integrity without requiring external services.

  • SQLite In-Memory: Uses StaticPool to override get_db() dependencies during testing.
  • Mocked External APIs: anthropic, prophet, and africastalking are mocked at the session level in conftest.py.
  • Comprehensive Coverage:
    • test_api.py: Validates Rev 2 schemas and telemetry structure.
    • test_security.py: Multi-role RBAC validation.
    • test_llm_advisory.py: Component C context building and graceful error handling.
    • test_anomaly_engine.py: Component B detection pipeline validation.
